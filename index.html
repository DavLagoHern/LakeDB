01 / Product

Built for real database work, not a perfect demo.

LakeDB keeps the context that usually disappears when you switch servers, editors and data grids. Each connection owns its workspace, open tabs and active database—so local, staging and production never blur together.

01

Independent workspaces

Open multiple connections at once. Every one preserves its SQL, tables, layout and session state.

02

Schema-aware SQL

Complete databases, tables, aliases and columns from live metadata, including PK and index context.

03

Typed table editing

Inspect and change large datasets with filtering, validation, conflict checks and safe rollback.

04

Operational tools

Explain plans, transactions, backups, restores, schema comparison and migrations stay close at hand.

05 / Daily details

The small interactions
that keep work moving.

LakeDB is designed around the moments repeated dozens of times a day: finding the right server, reaching an object, understanding a query and changing a row without losing confidence.

Cmd/Ctrl + P

Quick Open

Jump to connections, objects, open tabs, snippets and commands from one keyboard-first surface.

Connections

Switch without resetting

SQL

Context in the editor

Table data

Edit with an identity

Typed values, cell-driven filters and identity checks keep changes explicit before they reach the database.

06 / Yours by default

Local by design.
Personal where it matters.

Credentials stay in the operating system’s encrypted store. Connections support SSL, SSH, read-only mode and environment cues, while theme, density, font size and language adapt the workspace to you.

On your machine

No LakeDB account. No cloud credential service. No query telemetry.
